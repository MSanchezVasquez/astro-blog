Manejar operaciones asíncronas en JavaScript solía ser doloroso. Primero tuvimos callbacks, luego Promesas, y finalmente llegó `async/await` para salvarnos.

## ¿Cómo funciona?

Básicamente, `async` declara que una función va a manejar promesas, y `await` pausa la ejecución hasta que la promesa se resuelva.

```javascript
async function obtenerDatos() {
  try {
    const respuesta = await fetch(
      "[https://api.ejemplo.com/datos](https://api.ejemplo.com/datos)"
    );
    const datos = await respuesta.json();
    console.log(datos);
  } catch (error) {
    console.error("Hubo un error:", error);
  }
}
```

Advertencia: Solo puedes usar await dentro de una función marcada con async. Si intentas usarlo fuera, tendrás un error de sintaxis (aunque los módulos modernos ya permiten "Top-level await").

Patrón Paralelo
Un error común es hacer await secuencialmente cuando no es necesario. Si dos peticiones no dependen una de la otra, lánzalas al mismo tiempo:

```javascript
// ❌ Lento: Espera a una para iniciar la otra
const usuario = await getUser();
const posts = await getPosts();

// ✅ Rápido: Ambas inician a la vez
const [usuario, posts] = await Promise.all([getUser(), getPosts()]);
```
